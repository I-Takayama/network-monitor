# Network Monitor (C言語)

## 概要
C言語で実装したネットワーク監視ツールです。  
指定したIPアドレスとポート範囲に対して接続を行い、ポートの状態を定期的に監視します。

マルチスレッドによる高速スキャン、異常検知、CSV保存に加え、  
簡易HTTPサーバを実装し、監視結果をブラウザでリアルタイムに可視化できます。

---

## 開発背景

近年、サービスの可用性やネットワーク監視の重要性が高まっていると感じ、  
ポートの状態変化や異常をリアルタイムに把握できるツールを自作したいと考え、本システムを開発しました。

また、C言語によるネットワークプログラミングやマルチスレッド処理の理解を深めることも目的としています。

## 主な機能

- ポートスキャン（TCP connect）
- マルチスレッドによる並列処理
- 応答時間の計測
- CSVファイルへのログ保存
- 異常検知機能
  - 状態変化（OPEN⇄CLOSED）
  - 応答遅延
  - 連続失敗
- 簡易HTTPサーバの実装（C言語）
- JSON API提供
- ブラウザでリアルタイム監視表示

---

## 技術構成

- 言語: C
- OS: Linux / WSL
- 使用技術:
  - pthread（マルチスレッド）
  - socket API（TCP通信）
  - 簡易HTTPサーバ実装
  - JSON生成（手動）

## システム構成

[ポート監視スレッド]
        ↓
[ScanResult]
        ↓
[異常検知]
        ↓
[SharedStatus (mutex)]
        ↓
 ┌───────────────┐
 │ CSV保存       │
 │ Webサーバ     │
 └───────────────┘
        ↓
[ブラウザ表示]

## 工夫した点

- マルチスレッド化
    - ポートごとにスレッドを立てることでスキャンを並列化し、処理時間を大幅に短縮した

- 状態共有の設計
    - SharedStatus + mutex によりスレッド間で安全にデータ共有を実現し、競合状態を防止した

- 異常検知の実装
    - 前回状態との比較により、単なる状態表示ではなく「変化」を検知できるようにした
    - 連続失敗回数を管理し、一時的なエラーと継続的な障害を区別できるようにした
    - 応答時間のしきい値を設け、パフォーマンス劣化も検知可能にした

- C言語でHTTPサーバを実装
    - 外部ライブラリを使わず、socketのみでHTTP通信を実装し、内部動作を理解できるようにした

- リアルタイム可視化
    - JSON APIを自作し、JavaScriptで定期取得することでリアルタイム表示を実現した
